## 3. New file: STJ resolver module (`~SerdeStjResolverModule.g.fs`)

**Purpose:** expose a shared resolver instance and a `registerAll` function that integrates with `Serde.ResolverBootstrap`.

**Namespace:** `Serde.FS.SystemTextJson`  
**File name:** `~SerdeStjResolverModule.g.fs` (exact name is flexible, but keep the `~SerdeStj*.g.fs` pattern and same output folder as other STJ files).

**Content:**

```fsharp
// <auto-generated />
namespace Serde.FS.SystemTextJson

open System.Text.Json.Serialization.Metadata

module Resolver =

    /// Shared STJ resolver instance backed by GeneratedSerdeStjResolver.
    let resolver : IJsonTypeInfoResolver =
        GeneratedSerdeStjResolver() :> IJsonTypeInfoResolver

    /// Register this resolver with the Serde runtime bootstrap hook.
    ///
    /// Serde.ResolverBootstrap.registerAll is assumed to be a mutable global
    /// that other backends (like Serde.FS.Json) also use.
    let registerAll () =
        // The exact type of registerAll is:
        //   val mutable registerAll : (unit -> obj) option
        //
        // We store a thunk that returns the resolver boxed.
        Serde.ResolverBootstrap.registerAll <-
            Some (fun () -> box resolver)
```

> If `Serde.ResolverBootstrap.registerAll` has a slightly different type in the actual code (e.g., `unit -> unit` or `unit -> IJsonTypeInfoResolver`), adjust the lambda accordingly. The key invariant:  
> **calling `Resolver.registerAll()` must set `ResolverBootstrap.registerAll` to a nonŌĆæNone value that represents the STJ resolver.**

## 4. New file: Djinn bootstrap (`~SerdeStjBootstrap.g.fs`)

**Purpose:** satisfy DjinnŌĆÖs convention so it generates an entry point and calls into the STJ backend.

Djinn convention (already defined in `FSharp.SourceDjinn.Conventions`):

```fsharp
[<Literal>]
let ConventionBootstrapType = "Djinn.Generated.Bootstrap"

[<Literal>]
let ConventionBootstrapMethod = "init"
```

**Namespace:** `Djinn.Generated`  
**File name:** `~SerdeStjBootstrap.g.fs`  
**Content:**

```fsharp
// <auto-generated />
namespace Djinn.Generated

module Bootstrap =

    /// Called by FSharp.SourceDjinn's generated entry point via convention.
    ///
    /// This must exist with this exact name and signature:
    ///   type:   Djinn.Generated.Bootstrap
    ///   method: init : unit -> unit
    ///
    /// so that Djinn can discover it using the Conventions module.
    let init () =
        // Delegate to the STJ backend to register its resolver
        Serde.FS.SystemTextJson.Resolver.registerAll ()
```

This is all Djinn needs to treat STJ as a runtime backend.

## 6. Interaction with `Serde.FS.Json` (important behavior)

`Serde.FS.Json.SerdeJson` currently contains:

```fsharp
let private triggerBootstrap () =
    match global.Serde.ResolverBootstrap.registerAll with
    | Some _ -> ()
    | None ->
        let asm = System.Reflection.Assembly.GetEntryAssembly()
        if not (isNull asm) then
            match asm.GetType("Djinn.Generated.Bootstrap") with
            | null -> ()
            | ty ->
                match ty.GetMethod("init", System.Reflection.BindingFlags.Public ||| System.Reflection.BindingFlags.Static) with
                | null -> ()
                | m -> m.Invoke(null, [||]) |> ignore

let useAsDefault () =
    triggerBootstrap ()
    match global.Serde.ResolverBootstrap.registerAll with
    | Some f -> f()
    | None -> ()
    ...
```

Desired behavior after STJ changes:

- When STJ is present and its bootstrap is emitted:
  - DjinnŌĆÖs entry point calls `Djinn.Generated.Bootstrap.init()`.
  - That calls `Serde.FS.SystemTextJson.Resolver.registerAll()`.
  - `Serde.ResolverBootstrap.registerAll` becomes `Some ...`.
  - Later, when `SerdeJson.useAsDefault()` runs, `triggerBootstrap()` sees `registerAll` is already `Some` and **does nothing**.
  - JSON does **not** override STJ as the backend.

- When STJ is **not** present:
  - No `Djinn.Generated.Bootstrap` exists.
  - `ResolverBootstrap.registerAll` remains `None`.
  - `SerdeJson.useAsDefault()` calls `triggerBootstrap()`, which tries to find `Djinn.Generated.Bootstrap` and finds nothing.
  - JSON then sets itself as the default backend as it does today.

No changes are required in `Serde.FS.Json` for this behavior to workŌĆöSTJ just needs to set `ResolverBootstrap.registerAll` early via its bootstrap.

## 7. Expected end state

With `Serde.FS.SystemTextJson` referenced and the above files generated:

- **Build time**
  - PerŌĆætype STJ metadata is generated.

  - `Serde.ResolverBootstrap.registerAll` is set to STJŌĆÖs resolver thunk.
  - JSONŌĆÖs `triggerBootstrap()` sees `registerAll` is already set and does nothing.
  - STJ is the active backend; JSON is available but not the default.
  - SampleApp can use STJ endŌĆætoŌĆæend without codec errors.
