# 🌿 **Spec D — JsonTypeInfo Construction Pipeline**

## 🎯 **Purpose**
Define the deterministic pipeline that constructs a `JsonTypeInfo<'T>` for any type `'T` using:

- codec registry (Spec C)  
- property-level codec overrides  
- type-level codec overrides  
- generated metadata  
- STJ defaults  
- fallback rules  
- error behavior  

This pipeline is the single source of truth for how Serde.FS.Json decides:

- how to serialize `'T`  
- how to deserialize `'T`  
- how to apply codecs  
- how to override metadata  
- how to integrate with STJ  

It must be:

- deterministic  
- reflection‑free  
- AOT‑safe  
- Fable‑compatible  
- pure F#  
- easy for Claude to implement  

---

# 🌱 **1. Public API Surface**

### **1.1 Entry Point**
```fsharp
module JsonTypeInfoBuilder =
    val build<'T> : CodecRegistry -> JsonSerializerOptions -> JsonTypeInfo<'T>
```

This is the only public function.

---

# 🌳 **2. Pipeline Overview**

The pipeline runs in this exact order:

### **Step 1 — Check Codec Registry**
If a codec exists for `'T`:

- wrap codec into a `JsonTypeInfo<'T>`
- skip all remaining steps

### **Step 2 — Load Generated Metadata**
If generated metadata exists:

- load it  
- apply property-level overrides  
- return final `JsonTypeInfo<'T>`

### **Step 3 — STJ Fallback**
If no codec and no metadata:

- use STJ’s default `JsonTypeInfo<'T>`  
- apply property-level overrides  
- return final `JsonTypeInfo<'T>`

### **Step 4 — Error**
If STJ cannot produce metadata (rare):

- throw `SerdeMissingMetadataException`

This order is **strict** and must not be changed.

---

# 🌿 **3. Step 1 — Codec Registry Lookup**

### **3.1 Lookup**
```fsharp
match CodecRegistry.tryFind typeof<'T> registry with
| Some codec -> codecToJsonTypeInfo codec options
| None -> continue
```

### **3.2 Behavior**
- Type-level codec overrides everything  
- Property-level codecs still apply *inside* the codec’s JsonTypeInfo  
- Codec must be wrapped into a JsonTypeInfo using the adapter from Spec C

---

# 🌱 **4. Step 2 — Generated Metadata**

### **4.1 Discovery**
Generated metadata is exposed via:

```fsharp
val tryGetGeneratedTypeInfo<'T> : JsonSerializerOptions -> JsonTypeInfo<'T> option
```

This function is generated by Serde.FS.Json.SourceGen.

### **4.2 Behavior**
If metadata exists:

- load it  
- apply property-level overrides  
- return it  

If metadata does not exist:

- continue to STJ fallback

---

# 🌳 **5. Step 3 — STJ Fallback**

### **5.1 Construction**
```fsharp
let ti = options.GetTypeInfo<'T>()
```

### **5.2 Behavior**
- STJ fallback is allowed  
- property-level codecs override STJ behavior  
- type-level codecs do NOT apply here (they were handled in Step 1)

---

# 🌿 **6. Property-Level Codec Overrides**

### **6.1 Attribute**
```fsharp
[<JsonCodec(typeof<MyCodec>)>]
```

### **6.2 Resolution**
For each property:

1. Check for `[<JsonCodec>]` attribute  
2. Instantiate codec  
3. Wrap codec into a `JsonTypeInfo` for the property type  
4. Replace the property’s `JsonTypeInfo` with the codec’s version  

### **6.3 Precedence**
Property-level codec overrides:

- type-level codec  
- generated metadata  
- STJ defaults  

This is the highest precedence override.

---

# 🌱 **7. Type-Level Codec Overrides**

### **7.1 Attribute**
```fsharp
[<JsonCodec(typeof<MyCodec>)>]
```

### **7.2 Behavior**
If a type has a codec attribute:

- treat it as if it were registered in the registry  
- but only for that type  
- not for nested types  
- not for properties of other types  

This is implemented in Step 1 by checking attributes before registry lookup.

---

# 🌳 **8. Error Behavior**

### **8.1 Missing Metadata**
If:

- no codec  
- no generated metadata  
- STJ cannot produce metadata  

Throw:

```fsharp
exception SerdeMissingMetadataException of string
```

### **8.2 Invalid Codec**
If a codec throws during construction:

- wrap in `SerdeCodecException`

---

# 🌿 **9. Thread Safety**
The pipeline must be thread-safe.

- JsonTypeInfo construction is stateless  
- registry lookup is thread-safe (Spec C)  
- no global mutation occurs  

---

# 🌱 **10. Fable Compatibility**
The pipeline must:

- avoid reflection  
- avoid dynamic type loading  
- avoid Type.GetType  
- avoid runtime scanning  

All codec and metadata discovery is explicit.
